# TechFlow Task Manager

  - ## Descrição do Projeto
Este projeto simula o desenvolvimento de um sistema de gerenciamento de tarefas
para uma startup de logística, utilizando conceitos de Engenharia de Software e
metodologias ágeis.

O sistema permite criar, visualizar, atualizar e excluir tarefas, além de acompanhar
o progresso do desenvolvimento por meio de um quadro Kanban no GitHub.

  - ## Objetivo
Aplicar na prática:
- Metodologias ágeis (Kanban)
- Versionamento com Git e GitHub
- Organização de tarefas
- Testes automatizados
- Integração contínua com GitHub Actions

  - ## Escopo Inicial
- CRUD de tarefas
- Quadro Kanban
- Testes automatizados
- Pipeline de CI

  - ## Metodologia Ágil Utilizada
Foi adotada a metodologia **Kanban**, que permite:

- Visualização clara das tarefas
- Controle do fluxo de trabalho
- Facilidade na priorização
- Adaptação rápida a mudanças de escopo

O quadro Kanban foi criado utilizando a funcionalidade **GitHub Projects**, com as
seguintes colunas:
- To Do
- In Progress
- Done


  - ## Etapas do Desenvolvimento

O desenvolvimento do projeto foi dividido em etapas incrementais, seguindo os
princípios das metodologias ágeis:

1. **Planejamento Inicial**
   - Definição do escopo do projeto
   - Criação do repositório no GitHub
   - Organização do quadro Kanban

2. **Configuração do Ambiente**
   - Estruturação dos diretórios do projeto
   - Configuração do ambiente de desenvolvimento
   - Criação do código base

3. **Desenvolvimento do Sistema**
   - Implementação do CRUD de tarefas
   - Versionamento contínuo do código
   - Commits descritivos e frequentes

4. **Controle de Qualidade**
   - Criação de testes automatizados
   - Configuração do pipeline de integração contínua
   - Execução automática de testes com GitHub Actions

5. **Gestão de Mudanças**
   - Simulação de alteração no escopo do projeto
   - Atualização do Kanban
   - Documentação da mudança no README


  - ## Mudança de Escopo
Durante o desenvolvimento, foi identificada a necessidade de melhorar a priorização
das tarefas. Dessa forma, foi adicionada ao sistema a funcionalidade de **prioridade
de tarefas**, permitindo classificá-las conforme sua importância.

Essa mudança foi registrada no quadro Kanban e implementada respeitando os
princípios de adaptabilidade das metodologias ágeis.

  - ## Tecnologias Utilizadas
- Python
- Flask
- Pytest
- Git e GitHub
- GitHub Actions
